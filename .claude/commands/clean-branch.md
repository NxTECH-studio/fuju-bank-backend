# マージ済みブランチの整理

`develop` にマージ済みのローカルブランチを安全に削除する。

## 前提（このプロジェクト固有）

- **デフォルトブランチ**: `develop`
- **本番ブランチ**: `main`（保護ブランチ、直接 push 禁止）
- `main` / `develop` および現在チェックアウト中のブランチは **絶対に削除しない**

## 手順

1. **ベースブランチを特定する**
   - このリポジトリでは `develop` を使う
   - `git rev-parse --verify develop` で存在を確認（無ければ `main` にフォールバック）

2. **ベースブランチに移動する**
   - 現在のブランチが `develop` 以外なら `git switch develop`
   - 未コミット変更がある場合は中止してユーザーに確認

3. **リモートの最新状態を取得**
   ```bash
   git fetch --prune
   ```

4. **マージ済みブランチ一覧を取得**
   ```bash
   git branch --merged develop | grep -vE '^\*|^\s*(develop|main)\s*$'
   ```

5. **削除対象をユーザーに提示して確認**
   - 削除候補の一覧を表示
   - 「これらを削除してよいですか？」と明示的に確認

6. **承認されたら削除**
   ```bash
   git branch -d <branch>
   ```
   - `-d`（safe delete）のみ使用。`-D`（force delete）は使わない
   - 1 本ずつ実行し、失敗したブランチはスキップして理由を表示

7. **結果を表示**
   - 削除できたブランチ / できなかったブランチ / 理由をまとめて報告

## ルール

- `main` / `develop` は絶対に削除しない
- 現在チェックアウト中のブランチは削除対象から除外する
- マージ済みブランチのみを対象とする（`-d` を使い、`-D` は使わない）
- 削除前に必ずユーザーの承認を取る
- 未コミットの変更がある状態でブランチ切り替えをしない
